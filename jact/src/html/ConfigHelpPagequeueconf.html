Dokumentation : queueconf

 Queuegeneral

Global settings for call queues
persistentmembers[Boolean]Persistent Members Store each dynamic agent in each queue in the astdb so that when asterisk is restarted, each agent will be automatically readded into their recorded queues. Default is 'yes'.
queuesectionMap[Queuesection]


 Queuesection

call queue
sectionname[String]the section name
musiconhold[String]Musiconhold sets which music applies for this particula call queue (configure classes in musiconhold.conf)
announce[String]An announcement may be specified which is played for the member as soon as they answer a call, typically to indicate to them which queue this call should be answered as, so that agents or members who are listening to more than one queue can differentiated how they should engage the customer
strategy[String]ringall - ring all available channels until one answers (default) roundrobin - take turns ringing each available interface leastrecent - ring interface which was least recently called by this queue fewestcalls - ring the one with fewest completed calls from this queue random - ring random interface rrmemory - round robin with memory, remember where we left off last ring pass
servicelevel[Integer]Second settings for service level (default 0) Used for service level statistics (calls completed within service level time frame)
context[PbxContext]A context may be specified, in which if the user types a SINGLE digit extension while they are in the queue, they will be taken out of the queue and sent to that extension in this context.
timeout[Integer]How long do we let the phone ring before we consider this a timeout...
retry[Integer]How long do we wait before trying all the members again?
weight[String]Weight of queue - when compared to other queues, higher weights get first shot at available channels when the same channel is included in more than one queue.
wrapuptime[Integer]After a successful call, how long to wait before sending a potentially free member another call (default is 0, or no delay)
maxlen[Integer]Maximum number of people waiting in the queue (0 for unlimited)
announcefrequency[Integer]How often to announce queue position and/or estimated holdtime to caller (0=off)
announceholdtime[Boolean]Should we include estimated hold time in position announcements? Either yes, no, or only once. Hold time will be announced as the estimated time, or "less than 2 minutes" when appropriate.
announceroundseconds[String]What's the rounding time for the seconds? If this is non zero then we announce the seconds as well as the minutes rounded to this value
monitorformat[String]Calls may be recorded using Asterisk's monitor resource This can be enabled from within the Queue application, starting recording when the call is actually picked up thus, only successful calls are recorded, and you are not recording while people are listening to MOH. To enable monitoring, simply specify "monitor-format" it will be disabled otherwise. You can specify the monitor filename with by calling Set(MONITOR_FILENAME=foo) Otherwise it will use ${UNIQUEID}
monitorjoin[Boolean]If you wish to have the two files joined together when the call ends set this to yes
joinempty[String]This setting controls whether callers can join a queue with no members. There are three choices: yes - callers can join a queue with no members or only unavailable members no - callers cannot join a queue with no members strict - callers cannot join a queue with no members or only unavailable members
leavewhenempty[String]If you wish to remove callers from the queue when new callers cannot join, set this setting to one of the same choices for 'joinempty'
eventwhencalled[Boolean]If this is set to yes, the following manager events will be generated: AgentCalled, AgentDump, AgentConnect, AgentComplete (may generate some extra manager events, but probably ones you want)
eventmemberstatusoff[Boolean]If this is set to no, the following manager events will be generated: QueueMemberStatus (may generate a WHOLE LOT of extra manager events)
reportholdtime[Boolean]If you wish to report the caller's hold time to the member before they are connected to the caller, set this to yes.
memberdelay[Integer]If you wish to have a delay before the member is connected to the caller (or before the member hears any announcement messages), set this to the number of seconds to delay.
timeoutrestart[Boolean]If timeoutrestart is set to yes, then the timeout for an agent to answer is reset if a BUSY or CONGESTION is received. This can be useful if agents are able to cancel a call with reject or similar.

 Queuemember

Each member of this call queue is listed on a separate line in the form technology/dialstring. "member" means a normal member of a queue. An optional penalty may be specified after a comma, such that entries with higher penalties are considered last.
tech[String]the technology
dialstring[String]the dial string
