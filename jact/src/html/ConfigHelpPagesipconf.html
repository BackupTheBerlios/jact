sipconf

 Sipregister

Register with a SIP provider
user[String]is the user id for this SIP server (ex 2345)
authuser[String]is the optional authorization user for the SIP server
secret[String]is the user's password
host[String]is the domain or host name for the SIP server. This SIP server needs a definition in a section of its own in SIP.conf (mysipprovider.com).
port[Integer]send the register request to this port at host. Defaults to 5060
extension[String]the Asterisk contact extension. 1234 is put into the contact header in the SIP Register message. The contact extension is used by remote SIP server when it needs to send a call to Asterisk. See the example below. The default context extension is "s".


 Sipauth

Global credentials for outbound calls, i.e. when a proxy challenges your Asterisk server for authentication. These credentials override any credentials in peer/register definition if realm is matched. This way, Asterisk can authenticate for outbound calls to other realms. We match realm on the proxy challenge and pick an set of credentials from this list

 Sippeers

These variables can be configured for each SIP peer definition:

sectionname[String]Peer Section Name
accountcode[String]Users may be associated with an accountcode. See Asterisk billing
amaflags[Amaflags]Categorization for CDR records. Choices are default, omit, billing, documentation. See Asterisk billing
callgroup[String]
canreinvite[String]If the client is able to support SIP re-invites
context[PbxContext] Context in the dialplan for outbound calls from this client
defaultip[String]Default IP address of client host= is specified as DYNAMIC. Used if client have not been registred at any other IP adress.
dtmfmode[Dtmfmode]How the client handles DMTF? signalling
fromuser[String]Specify user to put in "from" instead of callerid (overrides the callerid) when placing calls _to_ peer (another SIP proxy)
fromdomain[String] Sets default From domain in SIP messages when placing calls _to_ peer
host[String] How to find the client - IP # or host name. If you want the phone to register itself, use the keyword dynamic instead of Host IP
incominglimit[Integer]Limits for number of simultaneous active calls for a SIP client
outgoinglimit[Integer]Limits for number of simultaneous active calls for a SIP client
insecure[String]Don't check host ip and port for peer (not user)
language[String]A language code defined in indications.conf - defines language for prompts and specific local phone signals
mailbox[String]Voicemail extension (for message waiting indications)
md5secret[String]MD5-Hash of ":asterisk:" (can be used instead of secret)
nat[Boolean]This variable changes the behaviour of Asterisk for clients behind a firewall. This does not solve the problem if Asterisk is behind the firewall and the client on the outside.
pickupgroup[String] Group that can pickup fellow workers' calls using *8 and the Pickup() application on the *8 extension
port[Integer] SIP port of the client
qualify[String]Check if client is reachable
restrictcid[Boolean] (yes/no) To have the callerid restricted -> sent as ANI; use this to hide the caller ID
rtptimeout[Integer]Terminate call if x seconds of no RTP activity when we're not on hold
rtpholdtimeout[Integer]Terminate call if x seconds of no RTP activity when we're on hold (must be larger than rtptimeout)
secret[String] If Asterisk is acting as a SIP Server, then this SIP client must login with this Password (A shared secret). If Asterisk is acting as a SIP client to a remote SIP server that requires SIP INVITE authentication, then this field is used to authenticate SIP INVITEs that Asterisk sends to the remote SIP server.
type[PeerType]Relationship to client - outbound provider or full client?
username[String]If Asterisk is acting as a SIP client to a remote SIP server that requires SIP INVITE authentication, then this field is used to authenticate SIP INVITEs that Asterisk sends to the remote SIP server. Also, for peers that register with Asterisk, this username is used in INVITEs until we have a registration.
musiconhold[String]Set class of musiconhold on calls from SIP phone. Calls to the phone require SetMusicOnHold cmd of higher priority (lower numerical value of priority) than Dial cmd in dialplan in order to set this class for the call. Calls have the MusicOnHold class set on a per call basis, not on a per phone basis, and making a call through any extension specifying SetMusicOnHold will override this value for the call.
callerid[String]the callerid

 Sipgeneral

The [general] section of sip.conf includes the following variables
autocreatepeer[Boolean]yes | no: If set, anyone will be able to log in as a peer (with no check of credentials; useful for operation with SER).
bindaddr[String]0.0.0.0 :IP Address to bind to (listen on)
bindport[Integer]5060 :port to bind to (listen on)
canreinvite[String]update | yes | no (global setting)
context[PbxContext] :This is the default context and is used when a endpoint has no context property. The context in secton of a endpoint is used to route calls from that endpoint to the wanted destination. The context body is located in extensions.conf.
defaultexpirey[Integer]120 :Default length of incoming/outoing registration
dtmfmode[Dtmfmode]inband | info | rfc2833 (global setting)
externip[String]200.201.202.203 :Address that we're going to put in SIP messages if we're behind a NAT
externhost[String]hostname.tld : (new in Asterisk 1.2.x)
externrefresh[String]xxx : Specify how often a hostname DNS lookup should be performed for the value entered in 'externhost' (new in Asterisk 1.2.x)
localnet[String]192.168.1.0/255.255.255.0 :local network and mask
fromdomain[String] : Sets default From: domain in SIP messages when acting as a SIP ua (client)
maxexpirey[Integer]3600 :Max length of incoming registration we allow
nat[Boolean]yes | no (per connection setting) Please note that in Asterisk 1.0.x nat can now have the values: no | never | route | yes.
notifymimetype[String]text/plain :Allow overriding of mime type in MWI NOTIFY used in voicemail online messages.
pedantic[Boolean]yes|no :Enable slow, pedantic checking of Call-ID:s, multiline SIP headers and URI-encoded headers
port[Integer]: Default SIP port. (this is not the port for Asterisk to listen. See bindport)
promiscredir[Boolean]yes | no Allows support for 302 Redirects; (No will redirect them all to the local extension returned in Contact rather than to that extension at the destination)
srvlookup[Boolean]yes|no :Enable DNS SRV lookups on calls
tos[String]: Set IP QoS? parameters for outgoing media streams (numeric values are also accepted, like tos=184 )
videosupport[Boolean]yes|no : Turn on support for SIP video
useragent[String]Allow the SIP header "User-Agent" to be customized
trustrpid[Boolean]yes|no : If Remote-Party-ID should be trusted
realm[String]my realm (Change authentication realm from Asterisk (default) to your own. Requires Asterisk v1.x)
relaxdtmf[Boolean]Relax dtmf handling
rtptimeout[Integer]Terminate call if xx seconds of no RTP activity when we're not on hold
rtpholdtimeout[Integer]when we're on hold (must be > rtptimeout)
progressinband[String]If we should generate in-band ringing always
usereqphone[Boolean]If yes, ";user=phone" is added to uri that contains a valid phone number.
compactheaders[Boolean]
sipdebug[Boolean]
checkmwi[Integer]
recordhistory[Boolean]
allow[CodecsSet]
disallow[CodecsSet]
