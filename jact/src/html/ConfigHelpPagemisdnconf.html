 Misdnconf

chan_misdn sample config general section: for debugging and general setup, things that are not bound to port groups
debug[Integer]set debugging flag: 0 - No Debug 1 - mISDN Messages and * - Messages, and * - State changes 2 - Messages + Message specific Informations (e.g. bearer capability) 3 - very Verbose, the above + lots of Driver specific infos 4 - even more Verbose than 3
tracefile[String]the big trace
trace_calls[Boolean]single call trace files set to true if you want to have them they depend on debug level
trace_dir[String]single call trace files set to true if you want to have them they depend on debug level
bridging[Boolean]set to yes if you want mISDN_dsp to bridge the calls in HW
stop_tone_after_first_digit[Boolean]Stops dialtone after getting first digit on nt Port
append_digits2exten[Boolean]Wether to append overlapdialed Digits to Extension or not
l1_info_ok[Boolean]set this to yes if you have jollys mISDN which sends correct L1 Infos
clear_l3[Boolean]set this to yes if you want to clear the l3 in case the l2 deactivates some environments have a flickering l2 which causes this option to damage active calls .. highly experimental
dynamic_crypt[Boolean]CRYPTION STUFF Wether to look for dynamic crypting attempt
crypt_prefix[String]crypt_prefix, what is used for crypting Protocol

 MisdnSections

name your sections as you which but not "general" ! the secions are Groups, you can dial out in extensions.conf with Dial(mISDN/g:extern/101) where extern is a section name, chan_misdn tries every port in this section to find a new free channel The default section is not a group section, it just contains config elements which are inherited by group sections.
context[PbxContext]define your default context here
language[String]Group Specific Language
nationalprefix[String]Prefixes for national and international, those are putted before the oad if an according dialplan is set by the other end.
internationalprefix[String]Prefixes for national and international, those are putted before the oad if an according dialplan is set by the other end.
rxgain[Integer]set rx/tx gains between -8 and 8 to change the RX/TX Gain
txgain[Integer]set rx/tx gains between -8 and 8 to change the RX/TX Gain
te_choose_channel[Boolean]Some telcos espacially in NL seem to need this set to yes, this is disabled by default, also in switzerland this seems to be important
early_bconnect[Boolean]wether to connect bchannel when getting proceeding
dialplan[Dialplan]dialplan options: 0 - unknown 1 - National 2 - International 4 - Subscriber This setting is used for outgoing calls
use_callingpres[Boolean]This is only for asterisk head and will result in only considering misdn.confs and misdn_set_opts callingpresentation informations if set to no. Otherwise asterisks callingpresentation overwrites misdn.confs settings.
always_immediate[Boolean]uncomment the following to get into s extension at extension conf there you can use DigitTimeout if you can't or don't want to use isdn overlap dial. note: This will jump into the s exten for every exten !
immediate[Boolean]uncomment the following if you want callers which called exactly the base number (so no extension is set) jump to the s extension. if the user dials something more it jumps to the correct extension instead
hold_allowed[Boolean]uncomment the following to have hold and retrieve support hold_allowed
pickgroup[String]Pickup and Callgroup
callgroup[String]Pickup and Callgroup
presentation[Presentation]Allows/Screens Callerid possible values: allowed not_screened be aware, if you set to allowed you need to set a correct callerid in the dialplan or set it here in the misdn.conf Some Telcos don't care about wrong callerids, others do !
echocancel[Boolean]this enables echocancelation, with the given number of taps be aware, move this setting only to outgoing portgroups !
echocancelwhenbridged[Boolean]this disables echocancellation when the call is bridged between mISDN channels
echotraining[String]Set this to 0 to disable echotraining
